Вопрос: Что такое PowerQuery?
Ответ: Power Query - это инструмент в Power BI, который позволяет вам очищать, изменять и анализировать данные.  Он предоставляет множество функций для работы с данными, таких как объединение таблиц, фильтрация, группировка и многое другое. 

В данном случае Power Query используется для:

* **Подключения к данным:**  Power Query может подключаться к различным источникам данных, таким как базы данных SQL Server, Excel, CSV файлы и т.д.
* **Просмотр данных:**  Power Query предоставляет возможность просматривать данные в виде таблиц, графиков и диаграмм. 
* **Профилирование данных:**  Power Query позволяет вам оценить качество данных, например, найти ошибки или несоответствия.

Вопрос: Как создать меру в DAX?
Ответ: В DAX для создания мер можно использовать следующие функции:

**1. `SUMX`:**  Вычисляет сумму значений по заданному столбцу или диапазону. 
   * Пример: `SUMX(Table, [Column])`

**2. `CALCULATE`:**  Определяет условия для вычисления мер. 
   * Пример: `CALCULATE(SUM([Sales]), FILTER(Table, [Date] >= DATE(2023,1,1)))`

**3. `AVERAGEX`:** Вычисляет среднее значение по заданному столбцу или диапазону. 
   * Пример: `AVERAGEX(Table, [Column])`

**4. `COUNTROWS`:**  Вычисляет количество строк в таблице. 
   * Пример: `COUNTROWS(Table)`

**5. `FILTER`:**  Определяет условия для фильтрации данных. 
   * Пример: `FILTER(Table, [Column] > 10)`

**6. `RELATED`:**  Вычисляет значение из связанной таблицы. 
   * Пример: `RELATED([Sales Table])`

**7. `IF`:**  Определяет значения в зависимости от условия. 
   * Пример: `IF([Column] > 10, "High", "Low")`

Query: Объясни разницу между функциями CALCULATE и FILTER с примером
Original Answer: ## Разница между `CALCULATE` и `FILTER`

**`CALCULATE`** и **`FILTER`** - это две мощные функции в DAX, которые используются для изменения фильтрации данных.  Различия между ними заключаются в том, что:

* **`CALCULATE`** изменяет контекст фильтрации **внутри** запроса. 
    * Это позволяет создавать сложные условия фильтрации, которые могут включать в себя другие функции DAX (например, `SUM`, `AVERAGE`, `COUNT`). 
    * `CALCULATE`  может принимать аргументы, которые представляют собой выражения или меры. 

* **`FILTER`** изменяет контекст фильтрации **на основе** запроса. 
    * Это позволяет фильтровать данные на основе определенных условий, заданных в запросе. 
    * `FILTER`  может принимать аргументы, которые представляют собой таблицы или столбцы.

**Пример:**

Представьте себе, что у вас есть таблица "Sales" с колонками "Region", "Country", и "Sales". Вы хотите вычислить сумму продаж в каждой стране. 

* **`FILTER`**  
    * `FILTER(Sales, Sales[Country] = "USA")` -  фильтрует данные только для страны США. 
    * В этом случае `FILTER` изменяет контекст фильтрации на основе конкретной страны. 
* **`CALCULATE`** 
    * `CALCULATE(SUM(Sales[Sales]), FILTER(Sales, Sales[Country] = "USA"))` -  вычисляет сумму продаж в каждой стране (США). 
    * В этом случае `CALCULATE` изменяет контекст фильтрации **внутри** запроса.

**Ключевые различия:**

* **`CALCULATE`** работает с выражениями и мерами, а `FILTER` работает с таблицей или столбцами. 
* **`CALCULATE`** может использовать другие функции DAX в своих аргументах, что позволяет создавать сложные условия фильтрации. 
* **`FILTER`**  предоставляет более простой способ фильтрации данных.

**Вывод:**

`CALCULATE` и `FILTER` - это важные инструменты для работы с данными в DAX. Выбор между ними зависит от конкретной задачи.
